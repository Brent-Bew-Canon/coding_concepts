jQuery

  • A JavaScript library that is a great help with DOM traversal
  • Faster than vanilla JS
  • Is being phased out in preference for other frameworks like React

Node.js

  • Allows you to write server-side code using JavaScript.
  • Cross-platform, open-source server environment
  • Back-end JavaScript runtime environment running on the V8 JavaScript Engine
  • Allows you to write command line tools for server-side scripting.

JSON

Asynchronous JavaScript

  • Technique that enables your program to start a potentially long-running task and still be able to be responsive to other events while that task runs. (Rather than having to wait for that task to finish first)
  • Once that task is finished, your program is presented with the result
  • The opposite is synchronous, which basically means that your program runs one thing at a time. If it takes 30 minutes to complete the first task, the program will be unable to respond or address anything else until after that task is completed

Object Oriented Programming (OOP)

  • Computer programming model that organizes software design around data, or objects, rather than functions and logic
  • OOP focuses on the objects that developers want to manipulate rather than the logic required to manipulate them
  • Examples of an object can range from physical entities, such as a human being who is described by properties like name and address, to small computer programs, such as widgets
  • Once an object is known, it is labeled with a class of objects that defines the kind of data it contains and any logic sequences that can manipulate it

Test Driven Development (TDD)

  • Software development process relying on software requirements being converted to test cases before software is fully developed
  • Tracks all software development by repeatedly testing the software against all test cases
  • This is as opposed to software being developed first and test cases created later
  • Process:
    • Add a test
    • Run all test with expectation that the new one fails
    • Write the simplest code to pass the new test
    • Run all tests again, all should pass
    • Refactor as needed
    • Repeat process

Jest

  • A JavaScript testing framework
  • Tests units of code to determine whether they are fit for use
